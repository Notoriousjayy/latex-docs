\section{Purpose}

This index summarizes the reusable PlantUML cloud architecture diagram library imported into the LaTeX documentation repository. The library is stored as source-controlled \repoPath{.puml} files and rendered into separate image-output directories for use in future architecture handouts.

\section{LaTeX Usage Pattern}

For \repoPath{pdflatex}, prefer PNG output unless the repository build has a supported SVG-to-PDF conversion path. A typical figure reference is:

\begin{verbatim}
\begin{figure}[H]
  \centering
  \includegraphics[width=0.95\linewidth]{cloud-architecture-diagram-library/png/cloud-arch-plantuml/03a_dynamic_scalability_sequence.png}
  \caption{Dynamic Scalability Architecture}
\end{figure}
\end{verbatim}

Use relative paths from the document directory. Do not use absolute filesystem paths.

\section{Rendering Output Locations}

Rendered output should be kept separate from source PlantUML files:

\begin{itemize}
  \item \repoPath{cloud-architecture-diagram-library/svg/<collection>/...}
  \item \repoPath{cloud-architecture-diagram-library/png/<collection>/...}
\end{itemize}

\section{Maintenance Rules}

\begin{enumerate}
  \item Preserve the imported collection structure.
  \item Keep \repoPath{.puml} source files version-controlled.
  \item Regenerate \repoPath{cloud-architecture-diagram-library-manifest.csv} when diagrams are added, removed, or renamed.
  \item Keep rendered images in dedicated generated-output folders.
  \item Do not embed raw PlantUML directly into LaTeX documents.
  \item Do not introduce vendor-specific cloud services unless already present in a source diagram.
\end{enumerate}

\section{Validation}

Before merging, verify that PlantUML rendering succeeds, this index compiles, and existing LaTeX documents still build through the normal repository workflow.
