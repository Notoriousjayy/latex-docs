    \midrule

    Concept &
    \begin{itemize}
      \item Identifying stakeholders' needs
      \item Exploring concepts, propose viable solutions
      \item Analyzing alternative architectures
      \item Preparing an architectural concept (such as when assembling a bid for a contract)
      \item Communicating between acquirers and developers as a part of contract negotiations
    \end{itemize}
    &
    \begin{itemize}
      \item Capturing the right stakeholders and concerns
      \item Support for proposal
    \end{itemize}
    \\[0.5em]

    Development &
    \begin{itemize}
      \item Refining system requirements
      \item Creating solution description
      \item Building system or systems
      \item Verifying and validating system
    \end{itemize}
    &
    \begin{itemize}
      \item Support for conformance to a normative specification
      \item Support for evaluation
      \item Support for development
      \item Support for input to generation and analysis tools
      \item Support for judging implementation conformance to architecture
      \item Support for project planning
    \end{itemize}
    \\[0.5em]

    Utilization &
    \begin{itemize}
      \item Operating the system to satisfy users' needs
    \end{itemize}
    &
    \begin{itemize}
      \item Support for help in tracking down operational errors
    \end{itemize}
    \\[0.5em]

    Support &
    \begin{itemize}
      \item Providing sustained system capability
    \end{itemize}
    &
    \begin{itemize}
      \item Support for system evolution in concert with the architecture and the associated business planning for evolution
    \end{itemize}
    \\
