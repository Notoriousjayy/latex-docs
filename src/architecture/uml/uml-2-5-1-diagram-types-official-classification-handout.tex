\textbf{Purpose.} This handout summarizes the official classification of Unified Modeling Language (UML) diagram types in UML 2.5.1.

\textbf{Official basis.} The Object Management Group (OMG) publishes UML 2.5.1 as a formal specification and identifies it as ``a specification defining a graphical language for visualizing, specifying, constructing, and documenting the artifacts of distributed object systems.''\cite{omg-about,omg-spec}

\section*{Classification Overview}
UML 2.5.1 defines \textbf{14 official diagram types}. These are commonly organized into:
\begin{itemize}[leftmargin=1.5em]
  \item \textbf{Structure diagrams} -- diagrams that emphasize the static organization of a system.
  \item \textbf{Behavior diagrams} -- diagrams that emphasize behavior, workflows, and changes over time.
  \item \textbf{Interaction diagrams} -- a specialized subset of behavior diagrams focused on message exchange among participants.
\end{itemize}

This yields the following count:
\begin{itemize}[leftmargin=1.5em]
  \item 7 structure diagrams
  \item 3 behavior diagrams
  \item 4 interaction diagrams
\end{itemize}

\section*{Interpretation Notes}
\begin{itemize}[leftmargin=1.5em]
  \item The term \textit{interaction diagram} is not a third top-level family separate from behavior diagrams; it is a recognized \textbf{subset of behavior diagrams}.\cite{omg-spec}
  \item ``Structure'' and ``behavior'' are the most useful high-level categories for organizing UML diagram selection in documentation and analysis.
  \item The official specification should be treated as the source of truth when naming or classifying diagram types.\cite{omg-about,omg-spec}
\end{itemize}
