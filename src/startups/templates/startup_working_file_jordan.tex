\entry{Startup Name:}{Smart Building Manager (SBM)}
\entry{Working Business Idea:}{Build a security-first smart-building platform that combines IoT operations, asset and energy visibility, cloud-enabled workflows, and automation for building operators.}

\begin{minipage}[t]{0.49\textwidth}
\entry{Problem:}{Building operators often manage disconnected systems, vendors, spreadsheets, and manual processes, making it difficult to control assets, energy, service quality, and risk from one place.}
\entry{Value Proposition:}{SBM provides a unified operating layer for smart-building data, workflow, and governance so teams can make faster decisions, automate routine work, and scale operations with better visibility.}
\entry{Current Stage:}{Validation / MVP definition (greenfield build)}
\end{minipage}\hfill
\begin{minipage}[t]{0.49\textwidth}
\entry{Target Customer:}{Small-to-mid-sized property owners, facilities teams, and smart-building operators that need better control of building assets, energy usage, and operational workflows.}
\entry{Revenue Model:}{Recurring SaaS subscription, implementation and integration services, and potential managed services or hardware margin over time.}
\entry{Current Focus:}{Narrow the first commercial wedge, define the MVP, and establish the product, data, and cloud foundation for launch.}
\end{minipage}

\textbf{Top 3 Immediate Priorities}
\begin{enumerate}
    \item Lock the initial beachhead use case and buyer so the first release solves one urgent, measurable problem.
    \item Define the MVP scope, core data model, and required integrations for the first production-capable version.
    \item Convert discovery work into design-partner or pilot conversations that validate demand, pricing, and onboarding needs.
\end{enumerate}
