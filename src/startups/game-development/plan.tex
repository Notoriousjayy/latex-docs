\entry{Startup Name:}{AETHERFORGE ENGINE}
\entry{Working Business Idea:}{Build a modular, professional game engine and tooling stack for developers who want one disciplined codebase that can target native desktop and WebAssembly from the start, with the long-term platform expanding into editor, runtime, and developer tools.}

\begin{minipage}[t]{0.49\textwidth}
\entry{Problem:}{Indie developers and small studios often face a tradeoff between convenience and control: managed engines can hide platform details and constrain low-level optimization, while custom engines and ad hoc frameworks make portability, tooling, and long-term maintainability expensive. Shipping to the browser adds another layer of complexity.}
\entry{Value Proposition:}{AETHERFORGE ENGINE offers a C-first, performance-oriented foundation with strict module boundaries, a thin platform layer, SDL3-based portability, OpenGL ES 3 / WebGL 2 rendering, and a shared native plus WASM build path. The goal is to give technical teams more control than off-the-shelf engines without forcing them to assemble every subsystem from scratch.}
\entry{Current Stage:}{Concept validation, architecture definition, and engine-foundation pre-MVP}
\end{minipage}\hfill
\begin{minipage}[t]{0.49\textwidth}
\entry{Target Customer:}{Technically capable indie developers, small game studios, and simulation-oriented builders who care about performance, portability, and owning more of their runtime stack---especially teams that want desktop and browser deployment from a shared codebase.}
\entry{Revenue Model:}{Initial phase: founder-built engine and tools with value measured through technical progress, developer adoption, and strong sample projects. Likely commercial paths: paid tooling, premium editor features, support subscriptions, commercial licensing, templates, training, and implementation services.}
\entry{Current Focus:}{Define the first commercial wedge, lock the engine core architecture, and build a vertical-slice foundation that proves the native plus WebAssembly workflow, rendering path, and starter tooling are viable.}
\end{minipage}

\textbf{Top 3 Immediate Priorities}
\begin{enumerate}
    \item Lock the first product wedge: decide exactly which developer and use case AETHERFORGE serves first, so the engine is shaped by a real buyer and project profile instead of broad ambition.
    \item Build the technical spine: core runtime, platform layer, rendering abstraction, asset pipeline starter, build presets, and a repeatable native plus WASM development loop.
    \item Produce a credible proof point: a polished sample project or vertical slice that demonstrates the engine can support real gameplay, iteration speed, and browser deployment without architectural drift.
\end{enumerate}
